Same job, different trade-offs.

Three head-to-head comparisons across the techniques covered in Section A, each scored on mechanism, complexity, strengths, weaknesses, and best-fit use case.

Stemming vs Lemmatization Shivam Sankpal

DimensionStemmingLemmatization
Working mechanismHeuristic suffix-stripping (e.g. Porter algorithm) without checking real-word validity.Dictionary + POS-aware lookup that maps a word to its true grammatical base form.
Computational complexityVery low — simple rule application, O(1) per word.Higher — needs POS tagging and a lexical resource lookup per word.
StrengthsFast, no dictionary required, good enough when speed matters more than precision.Linguistically accurate, output is always a real word, better for meaning-sensitive tasks.
WeaknessesProduces non-words ("studi", "univers"); can over/under-stem.Slower; requires additional linguistic resources and context (POS).
Suitable applicationsSearch engines, information retrieval at scale.Chatbots, QA systems, any task where output must be readable/correct.

TF vs TF-IDF Jayraj Sanas

DimensionTFTF-IDF
Working mechanismCounts a term's frequency within a single document only.Multiplies TF by IDF, factoring in how rare the term is across the whole corpus.
Computational complexityO(n) per document — a single pass to count terms.O(n) per document plus one corpus-wide pass to compute IDF; slightly higher overall.
StrengthsVery simple, cheap, easy to explain.Down-weights common words automatically, surfaces genuinely distinctive terms.
WeaknessesOverweights common words ("the", "is") that appear in every document.Needs the full corpus stats in advance; still ignores word order and meaning.
Suitable applicationsQuick keyword counts, simple frequency analysis.Search ranking, document similarity, classic text classifiers.

Word2Vec vs FastText Joint Analysis

DimensionWord2VecFastText
Working mechanism Learns one dense vector per whole word using CBOW (predict word from context) or Skip-gram (predict context from word). Represents each word as the sum of its character n-gram (subword) embeddings, then learns those subword vectors with the same CBOW/Skip-gram objective.
Computational complexity Lower memory and training cost — one embedding per vocabulary word. Higher memory and training cost — must store and combine many subword n-gram embeddings per word.
Strengths Fast to train, smaller model size, strong semantic relationships for common words in the training vocabulary. Handles out-of-vocabulary and rare words gracefully; excels on morphologically rich languages (e.g. Hindi, Finnish, Turkish) and misspellings.
Weaknesses Cannot produce a vector for a word it never saw during training; weak on rare/morphological variants. Larger model footprint; subword composition can blur meaning for short or unrelated-looking words.
Suitable applications English-heavy tasks with a large, stable vocabulary — recommendation, semantic search. Multilingual or low-resource-language NLP, social media text with typos/slang, spell-tolerant search.
